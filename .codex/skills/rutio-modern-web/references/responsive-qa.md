# Responsive QA

## Breakpoints
- 375px
- 430px
- 768px
- 1024px
- 1280px
- 1440px

## Checkpoints
- No horizontal scroll.
- Hero remains balanced.
- CTA text/button wrapping stays clean.
- Cards align and keep usable heights.
- Section spacing stays intentional.
- Forms remain readable and tappable.
- Footer links stay accessible.
- Placeholder cards do not overflow.

## Desktop Expectations
- Layout should not feel stretched.
- Max widths should look deliberate.
- Grids should align cleanly.

## Mobile Expectations
- Readable typography.
- Tappable controls.
- No dense paragraph blocks.
- No clipped or overflowing content.

## Validation Flow
- Use `npm run dev` for visual QA.
- Use `npm run build` for production safety when source code changed.
- Use `npm run lint` when relevant to changed code.
